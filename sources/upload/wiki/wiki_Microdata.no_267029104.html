Industrial Ecology Digital Lab : Microdata.no

Created by Candy Eugenie Charlotte Anquetil Ep Deck, last modified on 07.12.2022

This page is for documenting and guiding the access, development and use of the data on microdata.

About

Admission is open to approved research institutions, ministries, and directories.

  • Users may process and analyse all variables on the platform without limitations.
  • Users may share scripts and results as they like.
  • The database is continuously expanded and updated.
  • The institutions administer users themselves.
  • Privacy measures are integrated in the user interface. That is why no applications are needed. The solution has an anonymization interface with embedded privacy protection

microdata.no is a co-operation between Sikt – Norwegian Agency for Shared Services in Education and Research (former NSD) and Statistics Norway. The platform development is funded by The Research Council of Norway.

Background  

microdata.no offers access to register data with a set-up that makes it possible to use the data without any form of application.

This is an important simplification of data sharing for data users and data owners alike. The heart of the set-up is the platform’s metadata driven logic, opening for working with the data without seeing the individual records, and without the data leaving its protected data store. All results are anonymized on the fly with integrated privacy filters.

How to access?

In order to access the database, you need to reach out Rune Dahl.  You can simply ask him for access explaining you need to collect and aggregate data for your project. Your microdata account will be further linked to your personal ID-number. 

Documentation

Hosted at https://www.microdata.no/dokumentasjon/. The access requires an account (see above).

Please look also here : Ready,set, research

Prerequisites 

In order to be able to offer application-free access to sharp register data, microdata.no must meet certain conditions:

  • Users must work via metadata. They cannot view or download individual data.
  • The analysis tool must have built-in anonymization measures. Therefore, users cannot use their own statistics packages or libraries. They must use the tool provided in the service (which is continuously expanded).
  • All output must pass through a confidentiality filter, including noise. The methods are described in Appendix C in the user guide (PDF).


Exemple : Aquaculture in Trøndelag in 2015

In this study case, we are interested in knowing the age, nationality, sex, the level of qualification of people in Trøndelag working in aquaculture in 2015.
  • Connect to the variable database

    Always start by connecting to the database of variables that you will import into your local dataset in the next steps.
    As of today, there is only one available database: Statistics Norway's database.     It will eventually be possible to connect to databases that also contain data from other data owners.
    Use the following command in the command line to connect to Statistics Norway's database:

    require no.ssb.fdb:19 as db

    The last part of the syntax, “as db” is used to create a short name / alias for the database. You are free to choose other names if you wish.


  • Create an empty dataset

The next step is to create a local dataset that you will fill with variables from Statistics Norway's database. Use the following command and select an appropriate name for the
dataset (replace <dataset name> with a separate name):

create-dataset <dataset name>

In our example, we are using :

create-dataset aquaculture2015

Your dataset will first be empty. When you add variables with the command import, you will see in the upper left window that the dataset is filled with more and more variables, for each import command you execute



  • Define population

After creating a dataset, it is time to define the population we are going to study. This can also be done later, but it is recommended to do this as early as possible since the work
with data facilitation and analysis then becomes less resource / time consuming (the smaller the data set / population, the faster the response).
Populations are defined by:

- Retrieving variables to be used as population criterias
- Dropping units (individuals) that fall outside the criteria

We start by importing the variable BEFOLKNING_FOEDSELS_AAR_MND

import db/BEFOLKNING_FOEDSELS_AAR_MND as birthdate

We have now created a data set that consists of the entire Norwegian population.

Get to know the population by studying the age composition. For this, we first need to create a new age2015 variable based on the variable we have imported. Use the command generate and measure age per 2015.

What is imported is not only the year but also the month of birth (the format is YYYYMM). This is why we have to divide by 100 what is imported.

generate age2015 = 2015 - int(birthdate/100)

We can then look at the distribution of individual's age using the function histogram

histogram age2015,discrete



Note that the age range extends well over 100. The reason for this is that the population is not properly filtered. The dataset contains all the people in the
database who have ever lived, including dead people since the last 132 years.

We need a variable that indicates whether people are resident, dead, or emigrated to remove people we do not want to include in our analysis. For this
we use the variable BEFOLKNING_STATUSKODE and keep only the resident (regstat == '1') :

import db/BEFOLKNING_STATUSKODE 2015-01-01 as regstat2015

keep if regstat == '1'

This has direct consequence to diminish the number of the population associated with aquaculture2015, from 10 811 725 to 5 163 699.


We can now look again at the age distribution of the population living in Norway using the command histogram :

We can import other variables like the gender or the country of birth :


import db/BEFOLKNING_KJOENN as kjønn
import db/BEFOLKNING_FODELAND as country_of_birth



We can go further and look at the main employment(occup_code) or the associated NACE (Nomenclature of Economic Activities) number.

import db/REGSYS_YRKE_PUBL 2015-11-01 as occup_code2015
import db/REGSYS_VIRK_NACE1_SN07 2015-11-16 as nace2015

One has to be careful by importing variables as their name can change over time.

We can include also filter the population using the kommune or county  number. The counties group all the kommunes having the same 2 first digits. For example in 2015, Sør-Trøndelag and Nord-Trøndelag correspond respectively to counties 16 and 17.

import db/BEFOLKNING_KOMMNR_FAKTISK 2015-01-01 as kommune2015

generate county = substr(kommune2015, 1, 2)


Finally, we can access the level of education. The levels of education go from 0 to 9. Details/descriptions of the different levels are available in this document, from page 7.

import db/NUDB_BU 2015-01-01 as study2015

generate level = substr(study2015, 1, 1)

Now that all the variables are imported, we can start doing some analysis. Aquaculture corresponds to nace categories starting with 3 and trøndelag corresponds to counties 16 and 17.


Using the function tabulate, we can have the repartition of the population by selecting criteria (by filtering) :

We can know the level of education of people working in aquaculture (nace number start with 3) in trøndelag (county 16 and 17):

tabulate nace2015 level if ((county == '16'|county=='17')&(startswith(nace2015, '03' )))

We can be more precise in our demand and select only the men (kjønn=='1'):

tabulate nace2015 level if ((county == '16'|county=='17')&(startswith(nace2015, '03' )&(kjønn=='1')))


We can finally select only the norwegian (country_of_birth == '000')

tabulate nace2015 level if ((county == '16'|county=='17')&(startswith(nace2015, '03' )&(kjønn=='1')&(country_of_birth=='000'))
