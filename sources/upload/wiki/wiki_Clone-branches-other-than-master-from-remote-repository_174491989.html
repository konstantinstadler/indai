Industrial Ecology Digital Lab : Clone branches other than master from remote repository

Created by Unknown User (radekl), last modified on 11.11.2019

Often you would like to work on different branches other than the master branch locally. For instance, in the event that you decide to work together on a development branch of a remote repository.

After cloning the remote repository to your pc, the local branche in your repository will probably only be the master branch. You can double check this with the command git branch. To list all branches on the remote repository type:

 git branch -a

If you want to work on a branch other than the master branch, e.g. dev_branch, you will need to make a local copy of this branch. Assuming your remote branch can be referred to as origin, you can do this by typing:

 git co -b dev_branch origin/dev_branch
