Basic steps to contribute to projects hosted on the indecol Gitlab repository

We are using https://gitlab.com/ for all Indecol projects.

If you do not yet have account on gitlab please create one and contact anyone from IEDL personaly or by email: contact@iedl.no to add you to Indecol pojects. Make sure to and add this ssh key to your gitlab https://gitlab.com/profile/keys .

Once you are in you can start to contribute to projects. In case you have created the project and you are owner then you probably know what to do. Here we talk about situation when you want to contribute to indecol project.

Copy remote project to your local PC (this step is done only once for project)

  1. Go to web browser, login to https://gitlab.com/, find the name of the project and open it

  2. On the top there is button Fork and just next to it is box where you can see either https or ssh. Please make sure there is ssh

  3. Copy the url next to the ssh box

  4. On your PC open the command line or the Anaconda prompt (click for help)

  5. In the prompt navigate to folder where you want your local copy of the project (click for help)

  6. Run command:

    git clone url_copy

    where the url_copy is the copy from step #3. This will create your local copy of the project. Here you can do your changes and they keep on your local PC.

  7. Navigate to you local copy (step #6) of the project

    cd project

Once you are done with cloning the project to your local PC you do not need to do the steps again for this project.

Contributing to project (this is done every time you want to add/chage/fix anything in project)

  1. Open Anaconda promt and navigate to you local copy of project (last step from previous section)

  2. Navigate to master branch

    git checkout master

  3. Make sure you are up to date with remote repository (the one from https://gitlab.com)

    git pull

  4. Create branch where you will make changes or add new files

    git branch name_of_the_branch

  5. Switch to newly created branch

    git checkout name_of_the_branch

  6. Modify code, add new file[s], just develop whatever, the modified and/or new file[s] are on you local area

  7. Add file[s] from previous steps to git staging area

    git add file_name

    Repeat this for every modified and/or new file

  8. Once you are happy and you want to commit your changes run

    git commit -m 'short explanation of what you have done'

  9. Push your commited stuff to https://gitlab.com remote repository

    git push -u origin name_of_the_branch

After last step there is your branch where you have developed your stuff on the indecol gitlab repository. If you want this changes to be added to the main branch (master) to be visible by all other users of the project you have to create Merge request.

Merge request

  1. Open https://gitlab.com in web browser and navigate to the project.

  2. On the left navigation bar click Merge request and find green button (right side) New merge request and click on it.

  3. In the Source branch in Select source branch box select your branch (the one you have created in previous section).

  4. Target branch should be master

  5. Click on green button Compare branches and continue

  6. You can write some description of what and why you have done the changes. Select Assignee and/or Approvers if relevant.

  7. you can clik on Remove source branch when merge request is accepted. checkbox if you want to remove the branch automatically from the gitlab remote repository.

  8. Click on green button Submit merge request

That's it. The project maintainer[s] will review your request and accept it (or reject if it cause problems). Either way you will be notified.

Thank you for contributing!
