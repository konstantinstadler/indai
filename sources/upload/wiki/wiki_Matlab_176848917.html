MATLAB (matrix laboratory) is a multi-paradigm numerical computing environment.

You can find courses and tutorials at https://www.mathworks.com/products/matlab.html .

We propose to follow the coding style  matlabstyle.pdf works with GAMS .

Commenting style suggestion (Numpydoc-like)

Numpydoc is an excellent commenting standard for Python. Ideally, this should be emulated in Matlab scripts as well. Here's a quick guide for how to do that for Matlab functions:

Numpydoc-like comments are divided into sections. The first line is a short summary of the function. The rest of the sections are optional, and have headers when included. Here are headers applicable for Matlab: Deprecation, Extended Summary, Arguments, Output, Errors Thrown, See Also, Notes, Examples.

Example

If you prefer, you could use multiline comments instead of one percentage sign for every line:

function d = distance(point1, point2)
%{
Calculates Euclidian distance in any number of dimensions

Arguments
---------
point1 : vector of numerics
point2 : vector of numerics
%}

Header descriptions

Summary

A one-line summary that does not use variable names or the function name.

Deprecation

A section (use if applicable) to warn users that the object is deprecated. Contents should include information about when it will be removed, and the new recommended way of obtaining the same functionality.

Extended Summary

A few sentences giving an extended description. This section should be used to clarify functionality, not to discuss implementation detail or background theory, which should rather be explored in the Notes section below. You may refer to the arguments and the function name, but argument descriptions still belong in the Arguments section.

Arguments

Description of the function arguments, keywords and their respective types. List each argument on its own line with the argument name and the argument type, separated by a colon, and, optionally, a description on an indented line below. Here's an example:

Arguments
---------
argument_name : argument_type
    Description of argument `argument_name`
x : double
y : cell array
z : char


Output

Explanation of the output values and their types. Similar to the Arguments section, except the name of each output value is optional. The type of each output value is always required:

Returns
-------
int16
    Description of anonymous integer output value.

If both the name and type are specified, the Output section takes the same form as the Arguments section.

Errors Thrown

An optional section detailing which errors get raised and under what conditions:

Errors Thrown
-------------
MyComponent:noSuchVariable
    Variable not found in workspace


This section should be used judiciously, i.e., only for errors that are non-obvious or have a large chance of getting raised.

See Also

An optional section used to refer to related code. This section can be very useful, but should be used judiciously.

Notes

An optional section that provides additional information about the code, possibly including a discussion of the algorithm.

Examples

An optional section for usage examples.
