NTNU provides multiple solutions for data storage.

This can be a bit confusing, so here are our (Industrial Ecology) guidelines.

The guiding principle here is that you are not working alone but as part of a research team.

Thus, the Bus Factor applies - how  many team members have to suddenly disappear from your  project before the project stalls.



This naturally also applies to the infrastructure you use for storing your research data. If only you have access to that and you disappear (or just loose your password...) the project stops.

A further complication might be potential confidential data (personal/industrial data, exam results etc.). In general, most of our research data can be classified as internal, with the aim to get it open.

But there might be exception, in which case NTNU guidelines or the ones of the data providers apply (involve the IEDL in case you have such data).

With that in mind, these are the possible solutions for storing data:

Local Hard-disk

This is the least secure possibility to store your data. 

It should only be used as intermediate use e.g. for travel, or if you need fast access to do some calculations.

There is no backup in place (unless you manually set something up) and is also might be a security risk (check if your disk is encrypted).

NTNU Network drives

See here how to map these.

There are several around and should be the standard way to save your data. Note that the names (M, X, etc) are just names, we aim to keep them and the mount-point consistent but use the mount point name (the stuff after // ) if you communicate with orakel or other services which might not be aware of our IndEcol naming schemes. 

M-drive - \\home.ansatt.ntnu.no\USERNAME

This is your personal drive. Use it for personal documents (work contract etc.) not related to your research.

It is also used for local settings (e.g. desktop settings or conda environments on the windows server).

Should not be used for research data. Nobody as access except you.

T-drive - \\felles.ansatt.ntnu.no\ntnu\iv\ept

EPT wide admin drive, also used for teaching data. You might not have access if you are not involved in teaching.

Should not be used for research data.

X-drive - \\forskning.it.ntnu.no\ntnu\iv\ept\eksperimentell

The main drive for research data (unless you are working with "big" data in the TB range).

This is regularly backuped (see Backup Frequency). 

When you mount is a described you will find a folder \indecol - use this one (from there on it is restricted to only indecol employees - but keep the mountpoints as described above to make your scripts compatible with other users).

In \indecol you find

  • DATA: for common data and data intensive projects
  • Projects: for all other projects - make a folder with the project name in there
  • Software: generic scripts (mostly matlab and gams), rather use github instead
  • USERS: make a folder with your username in there for exchanging data/scripts not tied to a project

If a project goes into /DATA or /Projects is a bit arbitrary, talk with your supervisor or IEDL on how to set this up. If you just need something for a short term/personal project, use the /USERS directory. Generally, as soon as two people are working on something or one person for more then a month it should get a folder under /Projects.

If you have data in the TB range, talk with IEDL before (see also Big Data below). 


Data sharing outside IndEcol

For one-time sharing see Sharing large files .

As NTNU employee you have enterprise accounts at Box (1TB) and Onedrive (100 TB - theoretically).

There is also the possibility to have a Dropbox account, if it is really needed (mostly due to previous use in projects).

We used Box for all data exchange in the EXIOBASE projects and it worked without problems. 

Onedrive is very well suited for admin (office) documents but had problems with large files and rate limits for large data uploads (but this might have changed...).

Big data

Sigma2 - NIRD is the Norwegian Infrastructure for Big Data.

You and your supervisor need to apply for that.


Data dissemination

We recommend Zenodo - there is a Industrial Ecology and Quantitative Sustainability community and it is FAIR compliant.

Other potential solutions are Figshare, Dryad or OSF, prinipially any repository listed in the Registry of Research Data Repositories.

In addition, there is a NTNU Open Data repository, but with less quota and outreach then Zenodo.

If you need further help with Open Data Organization and Outreach you can also contact NTNU Research Data (or of course talk with us/IEDL).
