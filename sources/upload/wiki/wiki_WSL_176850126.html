Enabling Windows Subsystem for Linux

The Windows Subsystem for Linux (short: WSL) allows to run a Linux Kernel on a Windows 10 / Server.

WSL is available since build 14393 on 64 bit architecture (to check open the windows menu, type about and click “about your pc” - scroll down until you find the build information “OS build” and check also “System type”).

To enable WSL:

  1. Open the start menu (win) and search for “Turn Windows features on or off”

  2. Check the “Windows Subsystem for Linux” option

  3. Ok, wait and restart

  4. Open the microsoft store (search in the start menu for it)

  5. Search for “Ubuntu 18.04 LTS” (there are other Linux Distributions available, but this is the one we recommend as it is close to the version we use on our server and b/c we have some experience with this one- of course you can install others as well) and press “get”, then “install”

  6. Ubuntu Linux is now available from the start menu

  7. Start it for the first time to initiate the Linux installation, choose a username as password

You can install linux apps with the standard ubuntu apt-get. The first thing you should do is:

sudo apt-get update

followed by

sudo apt-get install build-essential

to install developer tools.

After installation your disks are available at /mnt/c (or any other disk letter you have available).

Note: You can work with the files on these drives from Windows and Linux, but you must not change the files from the rest of the linux system with any windows app (the whole directory structure is hidden anyway in windows).
