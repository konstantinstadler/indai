Changing password

Once in while it is good to change your password. To do so open terminal (which is done by ssh to server) and type

passwd


Accessing

From Linux

  • ssh username@IP

  • ssh -X username@IP (to use x window forwarding)

From Windows
  • Windows Terminal (recommended)  https://devblogs.microsoft.com/commandline/introducing-windows-terminal/

  • xming [recommended] (install from progdist/campus/X-Win32, read Readme.txt on how to install it)

SSH key-based authentication

For Ubuntu Server https://help.ubuntu.com/community/SSH/OpenSSH/Keys

in short (if you have already ssh pub/private keys from other servers/old redhat): in your .ssh dir type:

ssh-copy-id <username>@129.241.62.30

Open application

After you connect to server(using Xming f.e.) type in the console name of application. f.e.: matlab

Connect to samba shared folder

The local folders on the server are also accessible from your local computer.

This is similar to the ntnu network drives (internally we use the samba protocol to provide the folder).

Thus you can copy file in/from server using your file explorer on your local computer. Connect to it like this:

From Windows

Open Windows Explorer → right click on Network → select Map network drive Choose Drive → in folder box add \\129.241.62.30\share

You will probably need to Connect using different credentials if yours are not same. In the dialog popup set:

  • username: indecol\your_user_name

  • password: your_pasword

Enjoy!

From Linux

Either use nautilus, dolphin or any other file manager and in network use the url:

  • smb://129.241.62.30/share/

In you pop up window use:

  • username: your_user_name

  • workgroup: indecol

  • password: your_pasword

Connect to felles and EPT_eksperimentell (K-Drive/X-Drive)

You can connect from server to the NTNU network drives (e.g. X drive). This steps are done on the server (after you connect to server suing X-ming f.e.) There is 3 ways to connect to the felles and EPT_eksperimentell

  1. Most secure way as you do not expose your password. The downside is that the mounting will be lost after system rebooting and you will have to proceed step d again. Rebooting of system should be very rare and you will be informed ahead. (Except incidental power short cut of course)

    1. Open shell(terminal, console, Xming).

    2. Make sure you are at your home directory. Use command pwd Example:

      The shell output

      [your_user_name@iept1275 ~]$ pwd [your_user_name@iept1275 ~]$ /home/your_user_name
    3. Create mounting point. Lets call it eptex for example (choose whatever you like)

      The shell output

      [your_user_name@iept1275 ~]$ mkdir eptex

    4. Mount the X-drive to the eptex directory you just created. Use command

      sudo mount -t cifs -o uid=$USER -o gid=$USER -o user=$USER -o vers=2.0 //forskning.it.ntnu.no/ntnu/iv/ept/eksperimentell /home/$USER/eptex

      It will ask you for your NTNU password.

    5. Optional: You can copy-paste the line from d) in a text file in your home directory. Name the script something with '.sh', e.g. mount_x.sh . Then make the script executable with

      make exec
      chmod +x mount_x.sh

      When you then call the script with ./mount_x_sh it will first ask you for the your sudo password, and then for your ntnu password (which might be the same).

  1. Same as first option but in step 1d you can specify password directly. Replace PASSWORD with your NTNU password. (Not recommended - it stores your password to .bash-history, you need to manually remove your password from there)

    sudo mount -t cifs -o uid=$USER -o gid=$USER -o user=$USER -o vers=2.0,passwd=PASSWORD //forskning.it.ntnu.no/ntnu/iv/ept/eksperimentell /home/$USER/eptex

  2. In last option you will store your credentials in plain text. It will allow you to have automate connection even after system rebooting. Please consider this option carefully as the password is your NTNU password and in case the super admin (root) account will be exploited the password can be known to the hacker. If you still want to use this run commands

    printf "if grep -qs '/home/$USER/felles' /proc/mounts; then\n\t echo \"X-Drive is mounted.\" \n else\n\t sudo mount -t cifs -o uid=$USER -o gid=$USER -o user=$USER -o vers=2.0,passwd=PASSWORD //forskning.it.ntnu.no/ntnu/iv/ept/eksperimentell /home/$USER/eptex \nfi\n" >> ~/.bash_profile

    Replace PASSWORD with your NTNU passwordThis command will create entry in .bash_profile. The .bash_profile is run every time the system is boot or you can run it by typing 'bash' in terminal.

Note that if your password contains any of the characters

" ' ~ ! $ # & * ( ) | \

You will need to prefix those characters with “\” in the above command.

Running the Jupyter notebook

Launch the notebook from the server's terminal:

jupyter notebook --no-browser

From the local shell - set up the SSH tunnel:

ssh -L <PORT>:localhost:<PORT> <USER>@129.241.62.30


Follow the given url and token to access the server.
