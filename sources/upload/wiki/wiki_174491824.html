What is Anaconda

Anaconda (docs) is a free, easy-to-install package manager, environment manager, Python distribution, and collection of over 720 open source packages with free community support. Hundreds more open source packages and their dependencies can be installed with a simple conda install [packagename]. It’s platform-agnostic, can be used on Windows, OS X and Linux. The primary benefit of conda is the ability to maintain several separate environments, with possibly different version of Python, and ensuring that the package versions installed within each environment are compatible.


General

There are at least three different forms of Anaconda.  Which one is best depends on what you want to do.

  • Anaconda (Distribution) is the big install and comes with a base environment containing hundreds of standard packages for data science and such like.  It also comes with the "Navigator" GUI.  If you just want a standard collection of packages, this might be for you (
  • Miniconda is a lightweight version.  It installs conda and a base installation of Python but not a the full collection in Anaconda.  If you are likely to be working with your own custom environments then this is likely the preferred option.
  • Mamba (Github link here) is an optimised version of conda.  Critical aspects have been rewritten in C, e.g. the code to resolve package dependencies.  If you're likely to be changing package configurations from the default, setting-up your own environments, etc, then this will generally make the process much faster.

When installing, upgrading, or removing packages in a conda environment several "channels" are available (see docs here). There is the official default channel that is set, well, as the default. Then there is the popular conda-forge channel which is community-maintained and bioforge that is popular for bioinformatics. The default channel may be seen as more reliable but there may be some licensing restrictions in commercial use. Some packages may not be available in the default channel.
