Calling conda Python from Matlab (Windows)

  1. Firstly, be aware that the version of Python and Matlab have to be matched: see the list of possible matches here. So you may need to either change the version of Python in the conda environment you want to use and/or upgrade Matlab.

  2. Matlab will usually not find the correct Python binary without some help. You'll need to supply the path manually at the start of your Matlab script. You can find location of your Python as follows.
    1. If using cmd.exe then run where python.You'll likely see more than one entry: there's a Windows default Python ("...\Microsoft\WindowsApps\python.exe"), and there'll also be one or more entries due to your conda installation. Note the path for the appropriate conda Python.
    2. If using Powershall then run where.exe python (yes, the .exe has to be there). Then select in the same way as in a. above.
  3. Before you can use Python in Matlab, you'll have to run pyenv('Version', 'C:\path\to\your\python.exe'). Make sure to correctly replace the path in that command with what you obtained from step 2. If successful then you should see something like 
    ans = 
  PythonEnvironment with properties:

          Version: "3.10"
       Executable: "C:\Users\peterma\miniconda3\python.exe"
          Library: "C:\Users\peterma\miniconda3\python310.dll"
             Home: "C:\Users\peterma\miniconda3"
           Status: Loaded
    ExecutionMode: InProcess
        ProcessID: "17672"
      ProcessName: "MATLAB"

    The pyenv() command has to be run at the start of every command window session, or at the start of your script. More information can be found here.

  4. Other Matlab introductory docs can be found here.
