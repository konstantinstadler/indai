A compiled list of the most useful packages for using Python for scientific computing. This is a small sample of available packages, for an more extensive best of see the awesome Python list.

If you find any package missing, please add them with a link to the standard reference. To be included in the list, the package should either be part of the standard library, or installable by using pip or conda.

Standard Library

Some highlights from the Python standard library, for the full overview click here..

  • os: Interface to the file system, with works across Win and UX systems. Recently (since version 3.4), the package patlib (see below) provides an even better interface.

  • pathlib: Object oriented access to the underlying file system, works cross-platform. A short intro can be found here.

  • sys: Handling of the underlying system (e.g. username, OS), also includes sys.exit() for exiting Python within a program/script.

  • importlib: For reloading Python modules within a script (you need that when you develop a module and a script at the same time): importlib.reload(module_name)

  • collections: Extending the Python data types with named tuples, ordered dictionaries, â€¦

  • re: Regular expressions in python. See also our intro to regex.

  • smtplib: Sending emails from python, see the tutorial here.

  • itertools: Everything about looping and counting, e.g. avoid nested for loops by itertools.product

  • functools: Higher order functions, contains the decorator @lru_cache for caching function results - can be a massive speed up for repeated function calls.

  • logging: All logging functionality for Python. Basic tutorial.

  • sqlite: Build your own SQL database (if needed also just in memory) in python

  • json: JSON file encoder in python

  • datetime: manipulating dates and times

  • configparser: Parsing ini files in python

  • argparse: Parsing command line arguments for python scripts

  • multiprocessing: Run things manually (some packages like numpy run in parallel without any manual configuration needed) in parallel in python

Generic External Libraries

  • pyYAML: Access to yaml files

  • psycopg2: Access to PostgreSQL DB

  • pymongo: Access to MongoDB

  • xlwings: Interactive access to Excel Tables (for reading excel tables completely use Pandas).

  • flask: Building webpages in python.

  • pytest: Although Python contains built-in unit testing capabilities, pytest makes the whole process much easier.

  • mpi4py: Bindings for the Message Passing Interface (MPI) - multiprocessing for HPC

  • sqlalchemy: Python SQL toolkit and Object Relational Mapper

  • beautifulsoup: Python library for pulling data out of HTML and XML files

  • requests: HTTP library for Python.

  • nltk: platform for building Python programs to work with human language data


General Scientific Computing Libraries

Most of these are included in the Anaconda standard installation as described here.

  • Pandas: The go-to library for scientific computing in python.

  • numpy: Fundamental package for scientific computing with Python.

  • scipy library: Numerical routines for numerical integration and optimization

  • xarray: Multi-dimensional labeled arrays and datasets in Python. Access to netCDF files

  • sympy: Symbolic mathematics

  • matplotlib: Standard plotting library for Python, can be extended with seaborn (see below)

  • seaborn: Data visualization library based on matplotlib

  • h5py: Pythonic interface to the HDF5 binary data format

  • pygal: Interactive svg plots for Python

  • bokeh: interactive visualization library that targets modern web browsers for presentation

  • Dash: Python framework for building analytical web applications

  • cartopy: geospatial data processing and visualization in python

  • statsmodels: statistical models, tests, and statistical data exploration

Industrial Ecology, Climate Modelling and IAM Tools

  • brightway: LCA calculations in Python

  • pymrio: Multi-Regional Input-Output Analysis in Python

  • country_converter: Python package for converting country names between different classification schemes

  • pyhector: Python interface for the simple global climate carbon-cycle model Hector

  • pymagicc: Python wrapper for the simple climate model MAGICC

  • floweaver: View flow data as Sankey diagrams
