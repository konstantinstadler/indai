Industrial Ecology Digital Lab : Installation (conda): Anaconda, MiniConda, Mamba

Created by Peter Esmé Maxwell, last modified on 31.01.2023

Install Anaconda Single User (Windows)

  1. Download the Windows Installer from the relevant link on the download page.

  2. Once the file has downloaded, run it.

  3. Accept the license.

  4. Install as “Just me” (recommended).

  5. Select the path to install to. Usually, you can leave the suggested path, which should default to "C:\Users\username" where username is your username on the NTNU Windows computers; an alternative is something like "C:\Anaconda3" but it is more desirable for a single-user install to be under your own user directory. Check, however, that the path does not contain spaces or any special Unicode characters: any spaces of Unicode characters will typically cause the installation to fail, and you'd be likely to get problems later.

  6. On the next screen there are two check boxes.
    1. The "Add Anaconda3 to my PATH environment variable" should typically be left unticked because it can often cause problems/conflicts. Normally, if one runs the conda init command in step 9 below, then conda should integrate properly with other software (it will actually add entries into PATH anyway). There may be occasions when this doesn't work but they are unusual now.
    2. Ensure the "Register Anaconda3 as my default Python" is left ticked.

  7. Start the installation process.

  8. You don't need to install the additional packages suggested after the installation.

  9. In relation to item 6, you typically also want to let conda add itself to your shell.  Go to "Start Menu" → "Anaconda ... " → "Anaconda Powershell" and then run conda init.This will setup Anaconda into your shell and certain environment variables. This appears to be the preference for Visual Studio Code. If you open a standard Powershell after this step then you should see (base) in your command prompt, and you'll be able to call conda.

The official documentation is available here, which has screencaps that may be helpful.

Instructions on how to integrate the conda installed Python with Matlab can be found here.


Install Anaconda Single User (Linux)

  1. Download the latest installer from the page here, e.g. wget https://repo.anaconda.com/archive/Anaconda3-2022.10-Linux-x86_64.sh

  2. Run the installed, e.g. sh ./Anaconda3-2022.10-Linux-x86_64.sh 
  3. Accept the license. The suggested install location is usually $HOME/anaconda3 which is fine. Let the installer run.
  4. If the installer asks, let it run the conda init. If it doesn't ask, you will need to run $HOME/anaconda3/bin/conda init to add conda into your shell (it appends some lines into .bashrc), You'll have to close and restart your terminal session.


Install Miniconda Single User (Windows)

  1. Download the latest installer from the page here.

  2. The remaining instructions are pretty much the same as for the Anaconda Single User Windows install above. Ensure that conda is added to your shell: go to "Start Menu" → "Miniconda3" → "Anaconda Powershell Prompt", and then run conda init. Now when you open a shell, conda will be accessible.


Install Miniconda Single User (Linux)

  1. Download the latest installer from the page here, e.g. wget https://repo.anaconda.com/miniconda/Miniconda3-latest-Windows-x86_64.exe

  2. At time of writing, there is a bug in the installer (the "shebang" specifies sh but the script contains bash syntax).  So, one has to force execution with the correct shell, e.g. with the command bash ./Miniconda3-latest-Linux-x86_64.sh 
  3. Accept the license. The suggested install location is usually $HOME/miniconda3 which is fine. Let the installer run.
  4. If the installer asks, let it run the conda init. If it doesn't ask, you will need to run $HOME/anaconda3/bin/conda init to add conda into your shell (it appends some lines into .bashrc), You'll have to close and restart your terminal session.


Install Mamba Single User (Windows & Linux)

The recommended way to install Mamba is to to use their dedicated installer, which configures a Mamba install to use conda-forge as standard. The alternative way, which is detailed here, is to install Mamba as a package in an existing conda installation.

Don't bother trying this with the full Anaconda: without a lot of hinting, it'll spend hours trying to resolve the package dependences and still usually fail. It's really best to run this with a clean Miniconda install.

Also, at time of writing (Jan 2023), you cannot use the latest Linux Miniconda as it fails to resolve package conflicts: you must use the Python 3.9 Miniconda.  

  1. Run conda install mamba -n base -c conda-forge 
  2. Run mamba init then restart your shell session.
  3. Use mamba instead of conda.

For one of the several Github issues on the Mamba project about conflict issues between different mamba and conda versions, see https://github.com/mamba-org/mamba/issues/1735.
