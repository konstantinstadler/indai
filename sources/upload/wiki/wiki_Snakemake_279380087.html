Background

Snakemake is a workflow management system. It allows you to setup a potentially very complicated series of computation or data processing tasks, with automatic dependency handling and subsequent recalculation of only stale files. This can potentially be scaled-up to cloud or cluster installations without having to change your configuration.


Installation

There are installation instructions in Snakemake's documentation, which is the canonical resource. You also will almost certainly want to install Snakemake into it's own conda environment. As their documentation suggests, it's wise to use Mamba for this; if you're unsure of this, see our wiki page with instructions on how to setup Mamba..

As per the official Snakemake docs, the following will setup a conda environment with Snakemake installed in it and then activate it, using Mamba (this is assuming you have setup Mamba properly).

mamba activate base
mamba create -c conda-forge -c bioconda -n snakemake snakemake
mamba activate snakemake

Notably, the full installation cannot be performed on Windows; only a minimal install can be performed in Windows. You could setup a WSL (Windows Subsystem Linux) host to accommodate this, or perform your development on a remote server. Always ensure that you either work against a Git repository or keep a suitable backup of your work.

There is an extension for VSCode to properly highlight Snakemake's extension to the YAML syntax.

Reminder! You'll need to ensure the conda Snakemake environment has the all the packages you require installed in it.
