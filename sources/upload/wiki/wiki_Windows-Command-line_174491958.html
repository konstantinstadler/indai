First, don't be scared! Until not so long ago, the command line interface (CLI) was the only way to interact with a computer. Think of it like the interactive interpreter in Matlab - it is just a program that interprets text commands that you enter in the window.

If you are on MS Windows, you don't need to get proficient with the command line use. You should know, however, how to open it and some simple commands. If you are on Linux, you also might find the intro here useful in case you end up on a Windows machine - the commands are quite different to the Linux terminal.

Open the shell

To open the shell in Windows 10, click on the Windows symbol and type “cmd” or “command prompt”. You should than see an icon showing a little terminal window with C:\ in it. Click on it or press enter.

To open the shell at a specific location, navigate to the location in the File Explorer, press CTRL-l (or click on the adress bar) and type cmd. After pressing Enter the command shell opens at the location of the explorer. You can also go back to the File Explorer from the command line - just enter “explorer .” and a new File Explorer Window pointing to the location of the shell.

Changing the appearance

When the shell is open, click on the small icon on the upper left corner. This will open a menu with a entry “Properties”. You can change the font size, colour, opacity of the shell here.

Basic commands

After opening the shell it probably points to your user directory, thus showing something like

C:\Users\USERNAME>_

The C: indicates the drive you are on, the other parts the folders on this drive.

You can get a list of files in this directory by entering

dir

You will probably see a list of files (with a filesize) and folders (the latter indicated by <DIR>). If there are more files than fitting on one page, you can use

dir /p

which pauses when one screen is full.

On the top of the list you will find two entries: '.' and '..'. The single point represents the current directory, the '..' the underlying parent directory. To change to the parent directory, you use the change directory (cd) command:

cd ..

Again, you can have a look with 'dir'.

To go back, you use the directory name (which should be your username)

cd DIRNAME

You can also just enter the first couple of letters and use the <TAB> key to auto complete the rest.

You can create a new directory with

mkdir DIRNAME

and remove it again with

rmdir DIRNAME

You can only remove the directory if it is empty. Lets try that:

mkdir test 
cd test dir > content.txt 
cd .. 
rmdir test

You will get a message like “The directory is not empty”. So you have to first remove the file in there:

cd test 
del content.txt 
cd .. 
rmdir test

Alternatively, you can remove directory and files recursively with

rmdir test /S

Be careful with this command!

In the code example above, we use 'dir > content.txt' to write the output of the dir command to the file content.txt. You can do this with every command which prints to the shell using the '>' symbol. To copy the output to your clipboard, use the symbol '|', e.g. 'dir | clip'.

To change to a different drive, use the drive letter with ':'

x:

That's it for the beginning. There are, of course, many more commands. You can show them with help

and to get help to a specific command you can do either

help COMMAND

or

COMMAND /?

e.g.

dir /?

To exit the shell, do

exit

Other useful commands

cls … clears your screen

ipconfig … shows your ip address and other network infos

ping … test your internet connection, e.g. ping www.google.com

fc … compare two (text)files


Effective Use

- Use the explorer to navigate to the directory you want to work in
- Open the command line in that directory (cml in the address bar)
- Start the script
- Change directory structure - use the explorer: explorer.exe .
- Small edits: start notepad with notepad.exe filename


Batch files

- Small scripts to automate tasks
- Usually python is more useful
- But sometimes you need to do something quickly
- Just txt files with .bat ending
- notepad example.bat

Redirections


- `>`: redirect output to file
- `>>`: append output to file
- `<`: redirect input from file
- `|`: redirect output to input of another command

- Examples 
    - `dir > file.txt` 
    - `dir >> file.txt`
    - `dir | clip`
