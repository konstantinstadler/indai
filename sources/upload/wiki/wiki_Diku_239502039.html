Industrial Ecology Digital Lab : Diku

Created by Ruslan Zhuravchak, last modified on 25.02.2022
(Direktoratet for internasjonalisering og kvalitetsutvikling i høyere utdanningnning)-funded project for research and development of the pedagogical practices related to digital and web-supported forms of teaching and learning.
Involves two ongoing pilot courses
  • TEP4295 - Sustainable enginering
  • INGA2300 - Systems Engineering
Course materials are developed/made available primarily as Jupyter Notebooks containing:
  • training ground *-tg.ipynb with lecture notes, demos and the tasks for self-validation
  • assignment *-at.ipynb with homework instructions and the autograded tasks
The student-tutor interaction is facilitated by nbgrader and the e2x extension. This setup disseminates/collects/autogrades/delivers the feedback in JupyterHub which is deployed/managed/maintained by NTNU-IT at Openstack. 
The course materials in-production accessible at:
  • https://tep4295.apps.stack.it.ntnu.no/
  • https://ingx2300.apps.stack.it.ntnu.no/
A test-deployment at:
  • https://sandbox.apps.stack.it.ntnu.no/
A version-controlled project repository:

  • https://gitlab.com/rus.zhuravchak/tep4295

Course structure:

  • 0 The basics of python
  • 1 Life cycle inventory (LCI) analysis
  • 2 Mathematical framework for LCA
  • 3 Consists of two chapters:
    • Life cycle impact assessment (LCIA)
    • Life cycle footprint
  • 4 Material flow analysis (MFA)
  • 5 Energy analysis
  • 6 Economic assessment

Infrastructure

IAAS provided by the NTNU's OpenStack platform with (typically) ubuntu 20.04.3 LTS (Focal Fossa). The estimate of the resource needs depends on the number of students/users and the tasks.

 NTNU-IT makes these estimates and, additionally, may assist with:

  • deployment of the infrastructure
  • containerizing
  • installation of Jupyterhub, e2x, python environments, modules etc.
  • automating the data (grades) exchange with the learning management platform

Installation:

Assumes that there is a working JupyterHub setup with instructor roles enabled, nbgrader, relevant modules and extensions.

In the JupyterHub's courses directory, eg. ~/courses/ initiate the nbgrader's project which will create a course repository with the relevant configuration and dummy assignments:

~/courses$ nbgrader quickstart course_name

Remove the source directory (it will be substituted later):

~/courses$ rm -rf course_name/source/

Clone the gitlab project, copy the source directory to the new course and remove the clutter:

~/courses$ git clone --recurse-submodules https://gitlab.com/rus.zhuravchak/tep4295.git
~/courses$ cp -r tep4295/source/course_name/
~/courses$ rm -rf tep4295/

Since the materials under ./source/0-pybasics-tg rely on the external repository, this (external) repository is set as a submodule. In such case, clone with --recurse-submodules. 

The list of assignments from source directory should now appear in the formgrader's tab in the web interface. This lets edit the source files, generate/view/validate the student version of the assignment, release/collect/autograde these assignments, generate and release the feedback. Common batch-operations are also  available in nbgrader's CLI.

NB: "Generate" the student version of the assignment does not copy the notebooks that do not have any tasks in them by default. This applies to 0-pybasics-tg and requires to copy the *.ipynb files from source to release, e.g:

~/courses/course_name$ cp source/0-pybasics-tg/scientific-python-lectures/*.ipynb release/0-pybasics-tg/scientific-python-lectures/

Alternatively, add the *.ipynb to the not-ignored files (a config tweak).

NB: the changes in nbgrader_config.py may need a restart of the instructor's jupyterhub server (through admin panel).

Updating:

  • "Unrelease" the released assignments intended for updating, if any
  • Clone the gitlab project, substitute the old assignment files with the new ones in the appropriate directory under ~/courses/course_name/source/, as described above. In case of  tep4295, this procedure is simpler: ~/courses/tep4295$git pull
  • From the formgrader's tab in the browser - generate, validate and release the student version of the assignment
