Setting Indecol repository

The idea is to have master branch protected and to have develop branch where anyone from indecol and invited people can push to or/and merge other branches. This is simplest and fastest contribution flow. The maintainers of project then have right to push/merge to master branch. Usually it will be merging from develop branch. This flow will allow to have master branch to be always stable.

To protect master branch on gitlab see https://docs.gitlab.com/ee/user/project/protected_branches.html.

Allow merge and allow push should be set for master branch “Maintainers” (or choose names directly) and for develop branch choose “Developers + Maintainers”

To create develop branch see https://docs.gitlab.com/ee/user/project/repository/web_editor.html#create-a-new-branch.

To set develop branch as default see https://docs.gitlab.com/ee/user/project/repository/branches/#default-branch

The tutorial on how to contribute to this project is at Contributing to IndEcol internal projects
