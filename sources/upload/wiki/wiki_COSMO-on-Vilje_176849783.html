Industrial Ecology Digital Lab : COSMO on Vilje

Created by Unknown User (radekl), last modified on 11.11.2019
This is a tutorial for run COSMO-CLM on Vilje

In general, in order to run the model in vilje, we need to replace the several files to adapting the computer setting. This tutorial is about the chain mode for running the COSMO-CLM model.

ssh to vilje. Linux users use any shell, Windows users use xming from progdist

ssh -X -Y your_user_name vilje.hpc.ntnu.no

change directory to /work path. The work path can be shared with others.

cd /work/your_usr_name

Make directory called CCLM to store all CCLM related files and change to this directory

mkdir CCLM cd CCLM

Copy cclm-sp 2.4 package. It is located at xiangph's directory. Alternatively you can download it form COSMO web and copy to vilje if you wish.

cp /work/xiangph/CCLM/cclm-sp_2.4.tgz .

Unpack the package and move to subfolder cclm-sp

tar -xzvf cclm-sp_2.4.tgz mv cclm-sp_2.4 ./cclm-sp

Copy and unpack the supplementary data files for testing the starter package (the program wget needs to be installed on your system)

cd yourpath/cclm-sp/data ./get_sp_data.sh

Change into cclm-sp directory and initialize the package

cd cclm-sp init.sh

For the code to work three packages need to be compiled. These are cfu, int2lm, and cclm. If following this tutorial your current directory should be /work/your_usr_name/CCLM/cclm-sp. If this is not the case, then cd to this directory. Then proceed with the following steps. First we need to compile cfu.

cd src/cfu

Before compilation we need to load the fortran compiler and other modules

module load intelcomp/17.0.0
module load mpt/2.14
module load netcdf/4.4.1

Now we can proceed with making the cfu binary

make

Next we compile the int2lm code. Move to the int2lm folder located in src

 cd ../int2lm

In order to compile int2lm we need to change the contents in the Fopts file according to the new Fopts file int2lm_fopts.txt. Do not just copy the file and replace your old file, this might cause the “permission” issue.

Now we can compile int2lm, the binary will be in bin folder created upon initialization of the code above.

make

Next, change directory to the cclm source code.

cd ../cclm

Here too, we need to adjust the Fopts file. Change this file according to the file cclm_fopts.txt

Now we can compile cclm, the binary will be in bin folder created upon initialization of the code above.

make

Now We can navigate to chain direction to adapt the settings.

cd ../../chain/gcm_to_cclm/sp001

Change the settings of the files job_settings and subchain according to job_settings.txt and subchain.txt, respectively

!!!! You need to replace the xiangph to your own username on Vilje !!!!

We also need to change all the templates files

cd templates

according to the following files gcl2cclm_preptemp.txt, gcm2cclm_int2lmtemp.txt, gcm2cclm_cclmtemp.txt,  gcm2cclm_archtemp.txt, gcm2ccl_posttemp.txt, respectively.

!!!! Note: There is one place in int2lmtemp with my username which must be changed. And please also change the email address in all the files to your own address !!!!

Finally we can submit model to queue to vilje. Change to the folder sp001 and submit the job.

cd .. subchain start

If you want to see the status of your submitted job

qstat -f XXXXXXX.service2

You will also get emails when the jobs are finished.

The result nc files will be located at directory /work/your_usr_name/CCLM/cclm-sp/chain/work/sp001/post.

!!!Some comments!!!

Do not try to change the nodes and cpu setting at the first step since this causes to change the other parts of the codes.
