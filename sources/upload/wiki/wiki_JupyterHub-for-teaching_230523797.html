Deployment/Setup

  1. Obtaining the infrastructure at Openstack through NTNU hjelp. The (rough) estimate of the hardware specs depends on the number of users and the intended use.
  2. Setting up a standard instance using the guidelines (preferably ubuntu server, 18.04 or above and at least 2 GiB RAM). The flavor of  the VM must be at least m1.tiny to enable the installation of TLJH. 
  3. Installing TLJH on the (custom) server, customizing the users and the necessary permission.
  4. Installing the nbgrader and its extensions and the other packages if necessary. sudo -E does this for all the users.
  5. Getting a domain (either http://indecol.no/ or http://iedl.no/). We are uing Opalstack webapp provider where Digital Lab crew has access and can setup domain with DNS pointing to public IP address of Openstack instance
  6. Setting up https with Lets Encrypt (with the associated email admin@iedl.no).

Workflow

The "formgrader" tab enables to manage the courses, assignments, students and submissions. For more information on nbgrader workflow - refer to the documentation page.

The existing course repository can be cloned from e.g. Gitlab (Ubunttu server comes with git). Alternatively - transfer the materials from local machine using the "upload" in jupyterhub interface or through ssh.
