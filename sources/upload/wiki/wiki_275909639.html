Industrial Ecology Digital Lab : Uninstalling (conda): Anaconda, Miniconda

Created by Peter Esmé Maxwell, last modified on 31.01.2023

General

The uninstall process is mostly straightforward but a little care needs to be taken in relation to Anaconda's shell or environment integration.  Typically, problems in this respect are easier to resolve on Linux than Windows.  The uninstall instructions assume basic familiarity with using a command line / shell.

The official instructions are available in the Anaconda documentation but it appears to omit step(s) required to remove conda from the shell or environment.

The uninstall instructions assume a working conda installation.  Without that, more manual effort is required.

These instructions are for single user installs.  If you've installed multiuser then you should know what you're doing anyway.


Uninstall Windows (single user install)

  1. Run conda install anaconda-clean.
  2. Run anaconda-clean  (warning, this will remove all Anaconda related configuration!).
  3. You will likely need to undo the changes that conda init  made to your shell config during the installation process.  If you see the conda environment indicated in your prompt, e.g. (base), then this is the case.  Run conda init --reverse.
  4. If you requested alteration of the Window environment variables such as PATH when installing then you may need to edit this manually.
    1. In the window search on the taskbar, start typing "environment" until you see "Edit the system environment variables" (click to open).
    2. You should have a modal window titled "System Properties". Now, clock the "Environment Variables" button near the bottom of that window.
    3. You should now have a window titled "Environment Variables".  Under "User variables for", select "Path" and click the "Edit" button.  Remove any entries pertaining to Anaconda (carefully).
    4. Under "System variables", check that the "Path" variable is also clear of all references to Anaconda.
    5. Hit "Ok", etc, to confirm your changes.
  5. You can now remove the original install directory, i.e. the appropriate Anaconda3 directory.
  6. Any open terminals, etc, will have to be closed and reopened to effect the changes from step 3.
  7. You may have to manually clean-up Anaconda entries from your Start Menu.


Uninstall Linux (single user install)

  1. Run conda install anaconda-clean.
  2. Run anaconda-clean  (warning, this will remove all Anaconda related configuration!).
  3. You will likely need to undo the changes that conda init  made to your shell config during the installation process.  If you see the conda environment indicated in your prompt, e.g. (base), then this is the case.  Run conda init --reverse.
  4. You can now remove the original install directory, i.e. the ~/anaconda3  directory.
  5. Any open terminals, etc, will have to be closed and reopened to effect the changes from step 3.
