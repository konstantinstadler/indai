Industrial Ecology Digital Lab : Initiate git in existing folder and connect to remote repository

Created by Unknown User (radekl), last modified on 11.11.2019

Here are the steps for creating a git repository in folder where you already have a lot of code that is not version managed

In git bash, cd to the top-level directory where you have your code. E.g.

 cd 'C:\Users\[your_username]\[my_software_project]\'

Initiate git by

 git init

Add the files (e.g. all m-files in the directory) to the local git versioning.

 git add *.m

You can see that files have been staged for commit by typing: git status Commit those files

 git commit -m "initial commit message"

If not already done, create an empty remote repository in GitHub, Bitbucket, or Gitlab and copy the url to this remote repo. You than need to tell your PC to add the remote repo.

 git remote add origin INSERT_URL_HERE

Subsequently push your local code to the remote repo while setting up a master branch.

 git push -u origin master

You only have to do this once. Next commits will follow the normal git routine:

git add FILE_TO_ADD git commit -m "message" git push origin
